CH-DST-02 . family DST

DOT PLOT

two dots per category (two measures) on one scale. Needs: categories with two comparable measures (planned vs actual, min vs max).

comparisondistribution

Use when

  • two measures per category on one scale: planned vs actual, this vs that
  • Cleveland dot plot use cases

Do not use when

  • before/after change (CH-DEV-02 reads direction)
  • one measure (CH-RNK-05)

See instead

  • the two values are before/after: CH-DEV-02

Alternatives

  • CH-DEV-02 (change over time reading)

Rules

  • series identified by direct labels on the first row, never a legend
  • dot_a muted, dot_b primary, connector hairline
  • one highlight maximum

Never do this

  • legends
  • different scales per row
  • more than two dots per row

QA before delivering

  • series labeled on first row
  • highlight <= 1
  • unit once
  • source present

Data shape

titleaction title with a number
sourcebottom-left always
uniton first label
series_labels[name_a, name_b] labeled on the first row
categoriesarray of {label, values: [a, b], highlight?}
